# 01 — การวิเคราะห์ความต้องการ (Requirements Analysis)

## ภาพรวม
ระบบจัดการสต๊อกของในห้องเก็บของบ้าน (Home Stock Manager) สำหรับติดตามของกินสำรอง,
ของใช้, ยา, อุปกรณ์ ฯลฯ ว่ามีอะไร เหลือเท่าไหร่ เก็บที่ไหน ใครเบิกไป และของใกล้หมด/ใกล้หมดอายุหรือยัง

## ปัญหาที่ต้องการแก้
- ไม่รู้ว่ามีของอะไรอยู่บ้างและเหลือจำนวนเท่าไหร่
- จำไม่ได้ว่าเก็บของไว้ตำแหน่งไหนในห้อง
- ไม่รู้ว่าใครในบ้านเบิกของออกไป (ตามรอยไม่ได้)
- ซื้อซ้ำเพราะไม่รู้ว่ายังมี / ปล่อยของหมดอายุทิ้งโดยไม่รู้ตัว
- ของหมดกะทันหันเพราะไม่มีการเตือนล่วงหน้า

## ผู้ใช้งาน (Users)
สมาชิกในบ้านหลายคน ใช้เครื่องเดียวกัน เลือกชื่อตนเองจาก dropdown ก่อนทำรายการ
(ไม่มีระบบ login/รหัสผ่าน — เน้นความสะดวกในบ้าน) เพื่อให้ประวัติบันทึกได้ว่าใครทำรายการ

## ความต้องการเชิงฟังก์ชัน (Functional Requirements)

### FR-1 จัดการรายการของ (Item Management)
- เพิ่ม / แก้ไข / ลบ รายการของ
- ข้อมูลต่อชิ้น: ชื่อ, หมวดหมู่, ตำแหน่งเก็บ, หน่วยนับ, จำนวนคงเหลือ,
  จำนวนขั้นต่ำ (จุดแจ้งเตือน), บาร์โค้ด, วันหมดอายุ
- ค้นหาด้วยชื่อ/บาร์โค้ด และกรองตามหมวดหมู่/ตำแหน่ง

### FR-2 รับของเข้า (Check-in)
- เพิ่มจำนวนของรายการที่เลือก พร้อมระบุผู้ทำรายการและหมายเหตุ
- บันทึกลงประวัติเป็นรายการประเภท IN

### FR-3 เบิกของออก (Check-out)
- ลดจำนวนของรายการที่เลือก พร้อมระบุผู้ทำรายการและหมายเหตุ
- **ห้ามเบิกเกินจำนวนคงเหลือ** (จำนวนต้องไม่ติดลบ)
- บันทึกลงประวัติเป็นรายการประเภท OUT

### FR-4 ประวัติการเคลื่อนไหว (Transaction History)
- แสดงรายการ check-in/out ทั้งหมด เรียงตามเวลาล่าสุด
- กรองตามรายการของ / ผู้ใช้ / ช่วงเวลา

### FR-5 แจ้งเตือน (Alerts)
- **ของใกล้หมด:** จำนวนคงเหลือ ≤ จำนวนขั้นต่ำ
- **ของใกล้หมดอายุ:** วันหมดอายุอยู่ภายใน N วันข้างหน้า (ค่าเริ่มต้น 30 วัน, ปรับได้) รวมถึงของที่หมดอายุแล้ว

### FR-6 รายงาน (Reports)
- สรุปยอดคงเหลือทั้งหมด
- Export รายการของและประวัติเป็นไฟล์ CSV (รองรับ UTF-8 ภาษาไทย)

### FR-7 บาร์โค้ด (Barcode)
- ค้นหา/เลือกของด้วยบาร์โค้ดเพื่อ check-in/out ได้รวดเร็ว
- รองรับเครื่องสแกนบาร์โค้ด USB (ทำงานเหมือนคีย์บอร์ด พิมพ์รหัส + Enter)

## ความต้องการที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)
- **แพลตฟอร์ม:** Windows desktop, build เป็น `.exe` ไฟล์เดียว
- **ภาษา:** หน้าจอเป็นภาษาไทยทั้งหมด
- **ข้อมูลคงอยู่:** เก็บใน SQLite ไฟล์เดียว ข้อมูลอยู่ครบหลังปิด-เปิดแอป
- **ความถูกต้องของข้อมูล:** check-in/out ต้องเป็น atomic (จำนวนคงเหลือกับประวัติต้องตรงกันเสมอ)
- **ออฟไลน์:** ทำงานได้โดยไม่ต้องต่ออินเทอร์เน็ต

## นอกขอบเขต (Out of Scope — YAGNI)
- ระบบ login / สิทธิ์ผู้ใช้
- การติดตามแบบ batch/lot หลายวันหมดอายุต่อหนึ่งรายการ
- ซิงก์ขึ้น cloud / หลายเครื่อง
- สแกนบาร์โค้ดผ่านกล้อง (ใช้เครื่องสแกน USB แทน)
